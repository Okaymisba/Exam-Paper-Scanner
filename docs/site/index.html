An OCR-powered tool for teachers to upload exam sheet images, automatically extract student marks using GPT-4o vision, and export results to Excel — all from a clean web interface.

📦 What is this?

Overview

A full-stack web app that automates the tedious task of entering exam marks from physical answer sheets.

How It Works

1
Teacher signs up & gets approved
Create an account with your department info. An admin approves it before you can log in.
2
Create an exam
Enter subject name, code, number of questions, max marks per question, and CLO mapping.
3
Upload exam sheet images
Drag & drop or select JPG/PNG images of completed answer sheets. GPT-4o extracts the marks automatically.
4
Verify & save
Review OCR results, correct any errors, assign a roll number, and save the student's record.
5
Export to Excel
Download a formatted Excel file with all students, marks per question, total, percentage, and CLO breakdown.

Key Features

🤖
GPT-4o OCR
Uses OpenAI's vision model to read handwritten exam sheets with high accuracy.
📊

⚡ Getting Started

Quick Start

Go from zero to extracting your first exam marks in under 5 minutes.

1. Create Your Account

Go to the app's home page and click Sign Up. Fill in your:

  • Full Name — displayed in your profile and exports
  • Email — used for login (must be unique)
  • Password — minimum 6 characters
  • Department — e.g., "Computer Science"
⏳
Waiting for approval New accounts start as pending. You cannot log in until an admin approves your account. This usually takes less than 24 hours.

2. Log In

Once approved, enter your email and password on the login screen. Your session is saved in the browser — you won't need to log in again unless you explicitly log out.

3. Create Your First Exam

Click New Exam from the home screen and fill in the exam details:

1
Subject Name & Code
e.g., "Data Structures" and "CS-201". The code appears in Excel headers.
2
Number of Questions
How many questions are on the exam sheet. This determines how many mark columns appear.
3
Max Marks Per Question
Enter the maximum possible marks for each question. The OCR uses these to validate extracted values.
4
CLO Mapping (Optional)
Map each question to a Course Learning Outcome (CLO). Used to generate CLO-wise grade breakdowns in the export.

4. Upload & Extract

On the entry screen, drag & drop exam sheet images (or click to browse). For each image:

  • GPT-4o analyzes the image and fills in the marks
  • Review the extracted marks — edit any that look wrong
  • Enter (or confirm) the student's roll number
  • Click Save Student
✅
You're done! Repeat for all students, then click Export Excel to download your results file.

Roll Number Format

Roll numbers follow the pattern: PREFIX-YYNNN

  • Prefix — your program code, e.g., SE, CS, EE
  • YY — 2-digit batch year, e.g., 24 for 2024
  • NNN — 3-digit student number, e.g., 001

Example: SE-24001, CS-23015

💡
Tip The roll number prefix and batch year are set once during exam setup and auto-incremented. You only need to confirm the number part, not type the full string every time.

CLO Mapping

After entering the number of questions, you can optionally map each question to a CLO (Course Learning Outcome). This is used to generate CLO-level statistics in the Excel export, which is required for OBE-compliant reporting.

📌
What is a CLO? A Course Learning Outcome is a measurable statement of what students should know or be able to do after completing the course. Each question tests one or more CLOs.

Uploading Exam Sheets

The upload screen accepts JPG, PNG, or WebP images, up to 10 MB each.

Drag & Drop

Drag image files directly onto the dashed drop zone. You can drop multiple files at once — they will queue up and process one by one.

Batch Upload

Select multiple files by holding Ctrl (or Cmd on Mac) when clicking files in the browser dialog. Each image is processed sequentially.

⚠️
One student per image Each image should show a single student's answer sheet. Uploading a sheet with multiple students' answers in one image will produce incorrect results.

Reviewing Extracted Marks

After OCR completes, each question's marks appear in an editable table. The confidence score shows how certain the model is:

  • Green (>85%) — high confidence, likely correct
  • Yellow (50–85%) — medium confidence, check visually
  • Red (<50%) — low confidence, manual entry recommended

Click any mark cell to edit it. Totals and percentages update automatically.

Saving a Student Record

  1. Confirm or correct extracted marks
  2. Enter the student's roll number (auto-incremented from your exam config)
  3. Click Save Student
  4. The student appears in the table at the bottom of the screen
  5. Upload the next image

Deleting a Student Record

Find the student in the table and click the red delete icon (🗑) on their row. This permanently removes their data from the current exam session.

Exporting to Excel

Click Export Excel at the top of the entry screen. The download starts immediately. The Excel file contains:

Exam History

All your past exams are listed in the History screen. Click any exam to:

  • View all saved students and their marks
  • Add more students (continue the exam)
  • Re-export the Excel file

Each exam is associated only with your account. Other teachers cannot see your exams.

Admin Panel

The admin panel is only visible to accounts with the admin role. From there, you can:

  • Approve pending teacher accounts
  • Reject accounts that shouldn't have access
  • View a list of all registered users and their current status
🛑
Admin access is permanent Rejecting a teacher sets their status to rejected. They cannot log in. To re-enable access, the admin must change their status back to approved directly in the database (no UI for this yet).
🔬 OCR Guide

OCR Guide

Get the best accuracy from GPT-4o vision by formatting exam sheets correctly and understanding confidence scores.

How OCR Works

When you upload an image, the app encodes it to base64 and sends it to OpenAI's GPT-4o vision API along with a structured prompt that describes the exam layout and asks the model to extract the "Achieved Marks" row.

The model returns a JSON array of marks (one value per question). The app validates these against your configured max marks and returns a confidence score.

Exam Sheet Format

Your exam sheets should follow this standard table layout for best results:

Exam Sheet Table Layout 
┌──────────────────┬──────┬──────┬──────┬──────┬──────┐
│                  │  Q1  │  Q2  │  Q3  │  Q4  │  Q5  │
├──────────────────┼──────┼──────┼──────┼──────┼──────┤
│ Max Marks        │  10  │  20  │  15  │  25  │  10  │
├──────────────────┼──────┼──────┼──────┼──────┼──────┤
│ Achieved Marks   │   8  │  17  │  14  │  22  │   9  │
└──────────────────┴──────┴──────┴──────┴──────┴──────┘
✅
Best practices for exam sheets
  • Use a printed table (not hand-drawn)
  • Leave clear cell borders
  • Write marks in the center of each cell
  • Use dark ink (black or blue) on white paper

Image Quality Tips

  • Lighting: Shoot in good natural or even light. Avoid harsh shadows across the table.
  • Angle: Keep the camera directly above the sheet (top-down). Slight angles (<15°) are fine.
  • Resolution: 1 MP or higher is sufficient. Most phone cameras are more than enough.
  • Focus: Tap to focus on the mark table before shooting.
  • Background: Plain desk or table background works best.

Confidence Scores

Every OCR result includes a confidence percentage that indicates how certain GPT-4o was about the extracted marks.

High (>85%)
90%

Safe to accept automatically. Marks are clear and legible.

Medium (50–85%)
65%

Verify against the original image. Minor ambiguity in handwriting.

Low (<50%)
30%

Manual entry required. Image quality issue or illegible handwriting.

Common Failure Causes

IssueCauseFix
Wrong mark extracted Faint or smudged ink Retake photo with better lighting; edit manually
Mark shows as 0 or null Cell was left blank by student Correct is 0 — confirm or enter manually
All marks wrong Wrong row extracted (e.g., "Max" instead of "Achieved") Check if your sheet matches the expected layout
Confidence always low Pencil writing or very light ink Increase contrast when photographing; use pen ink on sheets
Numbers transposed Erasure visible on sheet Cross out errors clearly; rewrite next to them
GPT-4o API error OpenAI rate limit or temporary outage Wait 30 seconds and retry the upload

Known Limitations

  • Handwriting style: Very stylized or cursive numbers can confuse the model (e.g., European 7 with a crossbar, or crossed zeros).
  • Multiple erasures: Heavily erased cells with visible ghost numbers can cause wrong extractions.
  • Colored ink: Red or light-colored ink on white paper has lower accuracy than black/blue.
  • Non-standard layouts: If your sheet doesn't have a labeled "Achieved Marks" row, results will be unreliable.
  • Cost per image: Each upload costs approximately $0.001–$0.003 in OpenAI API credits depending on image size.

API Reference

All REST endpoints, request formats, and response schemas for the backend API.

Authentication

All protected endpoints require a JWT token in the Authorization header:

HTTP Header 
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

Obtain the token from the POST /auth/login endpoint. Tokens expire after the session — call login again if you receive 401.

Upload & OCR

POST /api/upload Upload exam sheet image and extract marks ▼

Accepts an exam sheet image, sends it to GPT-4o vision, and returns extracted marks with a confidence score. The request must be multipart/form-data, not JSON.

💻 Developer Guide

Developer Guide

Set up the project locally, understand the code structure, and run the tests.

Prerequisites

  • Python 3.10+
  • pip (comes with Python)
  • A Supabase project (free tier is fine)
  • An OpenAI API key with GPT-4o access

Local Setup

1
Clone the repository
bash
git clone https://github.com/Okaymisba/Exam-Paper-Scanner.git
cd exam-marks-app
2
Create and activate virtual environment
bash
python -m venv venv

# macOS / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
3
Install dependencies
bash
pip install -r requirements.txt
4
Create .env file
Copy the template and fill in your credentials (see Environment Variables below).
5
Run the app
bash
python app.py
# Visit http://localhost:5000

Running Tests

bash

pip install -r requirements-test.txt

# Run all tests
pytest

# Run with coverage report
pytest --cov=. --cov-report=term-missing

# Run only unit tests
pytest tests/unit/

# Run only integration tests
pytest tests/integration/

# Run E2E tests (requires running Flask server)
pytest tests/e2e/
📋
Test coverage target: 80% New features should include unit and integration tests. E2E tests cover critical user flows (signup, create exam, upload, export).
